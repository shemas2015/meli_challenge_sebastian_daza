3. Vulnerability Findings

SQL Injection — Product Search
ID: VSC-2026-001  |  CWE-89  |  CVSSv3: 9.8
Critical
EndpointGET /rest/products/search
Parameterq (query string)
MethodGET
CVSS Score9.8 (Critical)
Description

The q parameter in the product search endpoint is directly concatenated into a SQL query without sanitization or parameterized queries. An attacker can inject arbitrary SQL to bypass filters, extract data, or manipulate the database. The application discloses the raw SQLite error in the HTTP response, confirming active exploitation.

Request
GET /rest/products/search?q=' OR 1=1-- HTTP/1.1 Host: juice-shop:3000
Response
HTTP/1.1 500 Internal Server Error Content-Type: text/html; charset=utf-8 <h1>OWASP Juice Shop (Express ^4.22.1)</h1> <h2><em>500</em> Error: SQLITE_ERROR: incomplete input</h2>
Remediation

Use parameterized queries or an ORM with proper parameter binding. Never concatenate user input directly into SQL statements. Implement a global error handler to prevent raw database errors from reaching the client.

Insecure Direct Object Reference (IDOR) — Feedback Data Exposure
ID: VSC-2026-002  |  CWE-639  |  CVSSv3: 8.1
High
EndpointGET /api/Feedbacks
ParameterUserId (query string)
MethodGET
CVSS Score8.1 (High)
Description

The /api/Feedbacks endpoint returns all feedback entries from all users without any authorization check. Any unauthenticated user can retrieve user emails, ratings, and personal comments by supplying an arbitrary UserId. No authentication token is required.

Request
GET /api/Feedbacks?UserId=1 HTTP/1.1 Host: juice-shop:3000
Response
HTTP/1.1 200 OK { "status": "success", "data": [ { "id": 1, "UserId": 1, "comment": "...", "rating": 5, "User": { "email": "admin@juice-sh.op" } }, ... ] }
Remediation

Enforce object-level authorization on every request. Require a valid authentication token and validate that the requesting user is authorized to access the target resource. Apply authorization checks server-side, not only in the frontend.

Cross-Site Scripting (XSS) — DOM-Based via Product Search
ID: VSC-2026-003  |  CWE-79  |  CVSSv3: 6.1
Medium
EndpointGET /#/search
Parameterq (query string)
MethodGET
TypeDOM-Based XSS
CVSS Score6.1 (Medium)
Description

The Angular frontend reads the q parameter from the URL and injects its value directly into the DOM without sanitization. An attacker can craft a malicious URL containing an HTML/JavaScript payload that executes in the victim's browser upon visiting the link — no server-side reflection required. Confirmed via headless browser testing with payload <img src=x onerror=alert(1)>: the payload triggers an alert dialog.

Payload
GET /#/search?q=<img src=x onerror=alert(1)> HTTP/1.1 Host: juice-shop:3000
Evidence — Browser Execution
[!] XSS triggered — browser alert fired: "1" Detection method: dom_browser (Playwright headless Chromium) URL: http://juice-shop:3000/#/search?q=%3Cimg%20src%3Dx%20onerror%3Dalert%281%29%3E Result: VULNERABLE — JavaScript executed in browser context
Remediation

Sanitize URL parameters before injecting into the DOM. Avoid using innerHTML, document.write(), or jQuery's .html() with user-controlled data. Implement a Content Security Policy (CSP) header that restricts javascript: URIs and inline scripts. Use Angular's built-in sanitization pipeline and never bypass it with bypassSecurityTrustHtml().

Cross-Site Request Forgery (CSRF) — User Profile Update
ID: VSC-2026-004  |  CWE-352  |  CVSSv3: 6.5
Medium
EndpointPOST /profile
Parameterusername (body)
MethodPOST
CVSS Score6.5 (Medium)
Description

The profile update endpoint accepts POST requests using a session cookie (token) without any CSRF token validation. An attacker can trick an authenticated victim into changing their username by embedding a forged cross-origin form POST in a malicious page. The browser automatically sends the session cookie, and the server processes the request without verifying its origin.

Request (no CSRF token)
POST /profile HTTP/1.1 Host: juice-shop:3000 Content-Type: application/x-www-form-urlencoded Origin: http://evil-attacker.com Cookie: token=<victim_session_token> username=CSRF_DEMO
Response
HTTP/1.1 302 Found Location: /profile
Payload (auto-test)
{ "login_url": "/rest/user/login", "credentials": {"email": "admin@juice-sh.op", "password": "admin123"}, "form_data": "username=CSRF_DEMO" }
Remediation

Implement CSRF tokens (synchronizer token pattern) on all state-changing operations. Validate the Origin and Referer headers server-side. Set the SameSite=Strict attribute on session cookies.

Path Traversal — FTP Directory Confidential File Exposure
ID: VSC-2026-005  |  CWE-22  |  CVSSv3: 7.5
Medium
EndpointGET /ftp/legal.md
Parameterpath (URL path)
MethodGET
CVSS Score7.5 (High)
Description

The /ftp/ endpoint serves files directly without path sanitization or access controls. An attacker can use directory traversal sequences (../) directly in the URL path to read files outside the intended FTP directory. The server resolves traversal sequences and serves the resulting file without restriction, exposing confidential internal documents.

Payload
GET /ftp/../ftp/acquisitions.md HTTP/1.1 Host: juice-shop:3000
Response
HTTP/1.1 200 OK Content-Type: text/markdown; charset=UTF-8 # Planned Acquisitions > This document is confidential! Do not distribute! Our company plans to acquire several competitors within the next year. This will have a significant stock market impact...
Remediation

Validate and canonicalize all file paths before use. Resolve the absolute path server-side and verify it is contained within the allowed base directory. Restrict the /ftp/ endpoint to an explicit allowlist of public files. Apply authentication before serving any file from this directory.
